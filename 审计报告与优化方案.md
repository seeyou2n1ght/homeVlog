# HomeVlog 项目审计与优化方案 (Artifact)

## 一、 审计结果分析

经过对 `src/homevlog` 核心模块的全面审计，发现了以下潜在的安全漏洞、并发冲突隐患以及数据类型的兼容性问题：

### 1. 并发数据覆盖漏洞 (Critical)
- **位置**: `src/homevlog/utils/ffmpeg_tools.py` -> `FFmpegToolkit.cut_segments`
- **问题**: 切割的临时分段文件名被硬编码为 `part_{i:04d}.mp4`，且所有片段都输出到同一个 `temp_dir` (`outputs/temp_clips/`)。当未来流水线升级为多文件并发切割，或不同文件的流水处理过快时，极易发生旧文件被新文件的分段直接覆盖的风险，导致监控 Vlog 合成错误。
- **修复计划**: 将原始视频的文件名 (Stem) 加入前缀中，改为 `f"{Path(input_path).stem}_part_{i:04d}.mp4"`，实现临时文件的绝对隔离。

### 2. FFmpeg Concat 语法注入隐患 (Security Vulnerability)
- **位置**: `src/homevlog/utils/ffmpeg_tools.py` -> `FFmpegToolkit.merge_videos`
- **问题**: 生成 `concat_list.txt` 时使用了 `f.write(f"file '{abs_path}'\n")`。如果原始视频路径或时间戳中意外包含了单引号 `'`，将会破坏 FFmpeg concat 语法，导致合并任务彻底崩溃，甚至存在被利用执行任意参数的风险。
- **修复计划**: 对路径中的单引号进行安全的 Shell 转义：`abs_path = abs_path.replace("'", "'\\''")`。

### 3. SQLite 数据类型隐式转换风险 (Bug/Convention)
- **位置**: `src/homevlog/database/sqlite_manager.py`
- **问题**: 直接将 Python 的 `datetime.now()` 对象传入 SQLite 的 SQL 语句。SQLite 本身没有 DATETIME 原生类型（使用 TEXT 存储），如果未专门配置 sqlite3 的 `detect_types`，会依赖底层不可靠的隐式转换。
- **修复计划**: 显式调用 `datetime.now().isoformat()` 转换为标准 ISO 8601 文本格式入库。

### 4. 性能优化与规范建议 (Performance)
- **`scanner.py` 中的 `rglob`**: NAS 挂载盘上的 `rglob` 可能会阻塞主线程较长时间，尤其是在文件数量巨大的情况下。建议后续考虑换为非阻塞或更高效的 `os.scandir` 实现。由于改动会影响现有文件发现抖动逻辑，本次仅作标记。
